Наиболее важные теги

Не обязательно знать ВСЕ ТЕГИ, а лишь наиболее употребляемые, которые дадут возможность выполнить 80% работы. Остальные теги могут быть изучены по мере необходимости и решения той или иной задачи.

Итак, наиболее важные и часто употребляемые:

  • HEAD - шапка сайта. Между этими тегами размещаются все теги meta, этакие инструкции для браузера, скрипты, стили, тег head и другие, предназначенные для не для пользователей, а для браузера.
  • META - размещаются в шапке сайта.
  • BODY - тело сайта. Между этими тегами размещается ВСЁ, предназначенное для пользователя: текст, текстовая разметка, картинки, ссылки.
  • Title - заголовок сайта для поисковика, отображается в строке браузера.
  • H - текстовые заголовки сайта.
  • ul - для создания маркерованного списка.
  • ol - для создания нумерованного списка.
  • li - сам непосредственно список.
  • p - абзац. Между этими тегами располагается текст. У тега есть свои атрибуты, например, выравнивания текста
  • div - это контейнер, может содержать абзацы, объекты, может иметь свой стиль, более массивный блок по содержанию. Используется для придания внешнего вида целому блоку.
  • TABLE - тег для содания таблиц. Раньше таблицы использовались для создания сайта. Сейчас их предпочитают использовать для размещения конкретных данных на странице
  • tr - строка в таблице.
  • td - столбец в таблице.
  • a - тег ссылки. Используется с атрибутом href. Например, создадим несколько ссылок в виде нумерованного списка:
    1. Перейти на главную страницу сайта

    3. Подписаться на мой канал Ютуб
  • strong - выделение текста жирным.
  • em или i - выделение курсивом.
  • br - перенос строки.
  • сenter - выравнивание по центру.
  • font - раньше использовался для изменения характеристик текста, например, цвета. Сейчас для этих целей используют стили.
  • frame - делит окно на части.
  • iframe - между этим тегами можно вставить сторонний код для отображения информации с другого сайта. Например, видео из You Tube:

  • strike - перечеркнутый текст.
  • blockquote - выделение цитат.
  • hr - горизонтальная линия.
  • img - тег для размещения изображения на сайте. Например:

Итак, в данном случае на этой странице, подключив css стили и прописав правила, я отформатировал текст, задал шрифт, отступы, цвет и САЙТ ПРИОБРЁЛ уже приятный внешний вид. Такова роль стилей

Конечно, стили можно задавать и в разметке, точнее, так делали раньше до широкого применения стилей. Сейчас же все что касается оформления сайта - это задается в CSS. Это правило относится к процессу создания сайта. Верстальщик должен верстать по принципу - разметка (html) и стили (css).

Но насколько это правило "железное" для верстальщика, настолько же оно относительно для пользователя сайта. Поясню, что я имею в виду.

Когда пользователь пишет статью на сайте, например, то он пользуется инструментами форматирования текста, как в Worde, и таким образом, к тексту применяются атрибуты тегов html. В этом случае никак не задействуется CSS.

Захотел человек выделить текст жирным, нажал соответствующую кнопку в визуальном редакторе и текст стал жирным, а к коду добавился тег strong.

Получается, что ВСЕ, что касается самого КОНТЕНТА на сайте , к этой части можно применять атрибуты тегов.

А вот все, что касается оформления сайта, изначально, как ВСЕ будет работать и смотреться, то здесь ОБЯЗАТЕЛЬНО нужно использовать CSS.

Преимущество подключения к документу html таблицы стилей ещё и в том, что изменения в CSS влияют глобально на весь сайт. Таким образом, html - это гипертекстовая разметка сайта, a CSS - это стиль сайта.

Также на сайте можно использовать Java скрипты

Самый простой пример работы Java скрипта можно увидеть, кликнув по кнопке "Читать больше". Раскроется дополнительный скрытый текст.

Итак, HTML - это разметка текста, каркас сайта, подключенные файлы и другие команды в хедере. Это сам текст на сайте, контент, заключенный в блочную верстку

CSS стили - это оформление сайта, цвет, размещение элементов. С помощью тиблицы стилей можно созддать внешне 2 совершенно разных сайта, хотя гипертекстовая разметка будет одинаковой.

Java скрипты - это увеличение функционала сайта. Это уже элменты интерактивности человека на сайте. Кликнув по объекту он может запустить в действие ту или иную функцию. Как в данном случае, раскрылось скрытое под кнопокой содержание.
